SOLID Principles

Single Responsibility Principle

  1. The idea behind the SRP is that every class, module, or function in a program should have one responsibility/purpose in a program.

  2. As a commonly used definition, “every class should have only one reason to change”.

    1. Separation of concerns

  3. Advantages:
    1. Improved code readability: Reading code that abides by the SRP is easier to follow because it is focused and concise.

      1. A function that is doing too many things outside the realm of its responsibilities creates unnecessary entanglements that makes it harder to read and debug.
