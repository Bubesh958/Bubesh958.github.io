Programming Principles

SOLID Principles

Single Responsibility Principle

The idea behind the SRP is that every class, module, or function in a program should have one responsibility/purpose in program. As a commonly used definition, “every class should have only one reason to change”. Separation of concerns

Advantages:

  1. Improved code readability: Reading code that abides by the SRP is easier to follow because it is focused and concise.

Open Close Principle

Open for extension. Close for modification
